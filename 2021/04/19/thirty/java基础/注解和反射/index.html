注解和反射

注解

JDK 1.5 引入的新特性，开发人员通过注解告诉代码如何运行，然后配合反射得到注解的内容，决定如何运行；

Annotation 注解
meta-annotaion 元注解

自定义注解

几种常见的情况

定义注解注意事项
  • 注解参数后面需要带圆括号，eg: String name() ;
  • 当只有一个参数时，建议命名为value，使用的时候就不需要写参数名了 eg： @SuppressWarnings("all")
  • 注解的保留策略 Retention 直接用 runtime eg: @Retention(RetentionPolicy.RUNTIME)

元注解

元注解一共有4个，都是内置注解

4个元注解及其作用
@Target

包含一个参数 @Target(ElementType.METHOD)
包含多个参数 @Target({ElementType.METHOD, ElementType.TYPE})

可用的适用范围

@Retention

注解的保留策略，保留的三个阶段

SOURCE < CLASS < RUNTIME

source只保留在源码阶段 eg：@SuppressWarning(“all”)

class在class和source阶段都保留

runtime在三个阶段都保留，开发自定义注解都用这个

@Documented

标注的内容会出现在javadoc中

@Inherited

标注的内容会被子类继承
