Arrays.asList() 方法的使用和雷区

怎么使用

容易踩的坑

报异常UnsupportedOperationException

上个例子通过asList()得到的list在执行add()或者remove()时，就会抛出异常 java.lang.UnsupportedOperationException

以add()为例分析

我们先来看下asList()是怎么实现的

返回是ArrayList啊，ArrayList是有add()/remove()这些方法的，为什么通过asList得到的就不行呢？这是因为asList()返回的ArrayList和通过 new ArrayList<>()得到的ArrayList不一样

  • asList()返回的ArrayList并不来自java.util包，而是 java.util.Arrays.ArrayList，是Arrays自己定义的静态内部类

  • new ArrayList()得到的才是java.util.ArrayList，这才是我们常用的

二者都继承自AbstractList，区别在于java.util.ArrayList重写add()方法，前者直接使用父类AbstractList的add()方法，我们来看下父类中add()的实现

AbstractList中的add()方法直接抛出了UnsupportedOperationException()异常，也就是前面我们看到的现象。

数组值的怎么也变了

我们通过一段代码展示错误现象

现象：通过asList()方法把数组names转成List集合nameList，通过set()方法改变了集合中的值，这时候数组中的值也变了。

为什么？来看下Arrays中ArrayList的定义方式。

通过asList()获得List的集合，实际引用的就是我们传进去的数组，当我们执行set()操作时，就是对原数组进行操作。

奇怪的基本数据类型

当数组类型为基本数据类型时，通过asList()并没有得到想要的List 而是得到了List<int[]>

由此我们可以看出，该方法对于基本数据类型的数组支持并不好

总结

  • 如果想遍历list里的值，可以用Arrays.asList()；但是如果想新增或者 删除的时候，还是要new ArrayList<>()，然后把元素一个个添加进去

  • asList()最好使用对象型数组，不要使用基本数据类型的数组

    • 对象型数组，比如String，Integer，XxxVo
    • 基本数据类型 byte,short,int,long,float,double,boolean

  • 不支持 add()/ remove() /clear() 方法

  • 数组和生成的List值联动
