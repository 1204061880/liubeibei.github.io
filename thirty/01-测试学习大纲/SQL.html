SQL语句

​ SQL： 结构化查询语言，查询关系型数据库（MYSQL）

​ 特点：非过程性，一个语句对应一种结果

SQL分类

语言名称 英文缩写 作用 关键字
数据定义语言DDL DDL:DataDefinition Language 表操作 create / alter | TABLE / VIEW / INDEX
数据操纵语言DML DML:DataManipulation Language 表记录操作 insert / update / delete
数据控制语言DCL DCL:DataControl Language 库权限操作 grant / revoke / begin transaction commit
数据查询语言DQL DQL:DataQuery Language 表记录查询 select

数据表结构操作语句
创建表的语法

creat table 表名(列名 列类型(长度),列名 列类型(长度)…);

首先要选择往哪个数据库里存入表：use [数据库] 。例如：use mydb2;

修改表
  • 添加列 alter table 表名 add [列名] 列类型(长度) [约束];
  • 修改列 alter table 表名 modify [列名] 类型(长度) [约束];
  • 删除列 alter table 表名 drop [列名];
  • 修改列名 alter table 表名 change [旧列名] [新列名];
  • 修改表名 rename table 表名 to [新表名];

语法：drop table [表名]

数据表记录操作语句 ☆☆☆
添加记录
  • insert into 表名 (字段1,字段2,….) values (值1,值2,….)
  • 省略前面的字段 insert into 表名 values (值1 ,值2 ,….)
  • 以上两个方法插入的是所有字段，现在选择个别的字段插入 ，不插入的字段使用默认值
  • 注意事项
    • 字符串和日期类型 需要加上单引号
    • 插入的数据不能超过设定长度
    • 插入的时候值的位置要与字段的位置对应
  • 插入中文记录
    • insert into employee values (4,’李四’,’man’,’1992-01-01’,’2014-05-08’,’worker’,8000,’goodworker!’);
    • 如果插入中文时候报错，因为没修改字符集 ERROR1366 (HY000): Incorrect string value: ‘\xD5\xD4\xC1\xF9’ for column ‘name’ atrow 1
    • 解决办法：
      • 一种临时：set names gbk;
      • 第二种：修改mysql配置文件，在mysql的安装路径下的文件 my.ini 找到 [mysql]改为： default-character-set=gbk

​ 查看是否插入成功：select * from employee;

修改记录

update 表名 set 字段名=值,字段名=值 [where 条件]

删除记录

delete from 表名 [where 条件];

  • 如果没有条件 默认删除整个表 中的所有记录。
  • 按照主键删除，两种方式
    • TRUNCATE TABLE相当于把整个表删除了，然后重新创建了一个表，不可恢复。
    • DELETE FROM 把表逐行删除

数据库的备份和还原
备份数据库

在cmd界面中敲入命令 mysql dump -u root -p 数据库名 > c:\backUp.sql

还原数据库

1、在cmd界面中敲入命令 mysql -u root -p 数据库名 < backUp.sql

2、创建一个数据库.

3、 进入到数据库中 执行 source c:\backUp.sql

​ char和 varchar区别?

char属于是定长. varchar可变长度 例如： char(8) 输入一个4个字符的字符串，就会使用空格补全其他的4个长度

varchar(8) 输入4个字符就相当于是4个字符，varchar的长度定为4

​ datetime和 timestamp 区别? 他们都是既有日期又有时间，区别在于没设定时间时timestamp可以自己存入系统的时间.
